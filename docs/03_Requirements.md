# Product Requirements for RoomLight

This document lists Functional and Non-Functional requirements for RoomLight.

| REQ-ID | Requirement Description | Type | Priority |
| :--- | :--- | :--- | :--- |
| **REQ-001** | Dashboard allows admin to create and save light settings to a profile. | Functional | Must |
| **REQ-002** | Admin can sync the profile to all connected lights with one click. | Functional | Must |
| **REQ-003** | Dashboard a list of all rooms and light status in the room. | Functional | Must |
| **REQ-004** | Guests can control room specific lights from a dashboard with room-specific permissions. | Functional | Should |
| **REQ-005** | Admin can create groups for areas or rooms (floor 1, bathrooms, etc) to sync lights to specific rooms. | Functional | Should |
| **REQ-006** | Dashboard requires a username and password to log in. | Non-functional | Must |
| **REQ-007** | Every room saves settings locally so lights don't stop working if network is down. | Non-functional | Must |
| **REQ-008** | System supports at least 500 rooms at once. | Non-functional | Should |
| **REQ-009** | Dashboard is easy to use with clear buttons that require no advanced instruction. | Non-functional | Must |
| **REQ-010** | When syncing lights change should apply to all rooms within a minute. | Non-functional | Should |
